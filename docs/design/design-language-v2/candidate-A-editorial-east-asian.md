# 候选方向 A：编辑级东亚（Editorial East Asian）

> 提案者：CTO + Tech Lead（视觉总监角色）
> 日期：2026-04-28

## 1. 北极星（一句话定位）

**一本你愿意每天翻一页的私人健康内刊，不是健康 app。**

## 2. 心智 Mood Keywords

克制、留白、编辑级排版、单色基底、一抹强色、有机不对称、高级感、哑光质地、毛笔书法节奏、杂志栅格、慢节奏、沉静、纸感、宋体优先

## 3. 色彩系统

色彩哲学：**90% 单色 + 10% 一抹强色**。让强色稀缺到每出现一次都像句号。

### 3.1 Light Mode

| 角色 | 名称 | Hex | 灵感 |

## 4. 字体系统

**核心立场**：标题用衬线（宋体/Spectral），正文用人文无衬线（思源黑体 / Inter）。这就是和 v1（Plus Jakarta Sans 圆润 + Be Vietnam Pro 中性）最根本的分手。

| 角色 | 西文 | 中文 | 日文 | 理由 |
|---|---|---|---|---|
| Display / Headline | **Spectral SemiBold** | **思源宋体 (Source Han Serif) Medium** | **Noto Serif JP Medium** | 宋体的撇捺起笔即"毛笔节奏"。Spectral 是 Google 出品的免费杂志衬线，与思源宋体在字宽与黑度上对得上 |
| Title | Spectral Regular | 思源宋体 Regular | Noto Serif JP Regular | |
| Body | **Inter Regular** | **思源黑体 (Source Han Sans) Regular** | **Noto Sans JP Regular** | Inter 是事实上的"编辑级中性体"（Linear、Vercel、Stripe 都在用），与思源黑体灰度匹配最佳 |
| Label / Caption | Inter Medium（tracking +60） | 思源黑体 Medium（tracking +40） | 同 | 标签字间距放宽是杂志页眉惯用语法 |
| 数字 | **JetBrains Mono Light**（仅在表单/数据可视化） | — | — | 等宽数字，给 PK 曲线和血检数值"实验室签证"感 |

### Type Ramp

| Token | Size / Line-height / Tracking | 用途 |
|---|---|---|
| `display-xl` | 40 / 52 / -0.5 | Today 顶部问候、Onboarding 大标题 — 一屏只有一个 |
| `display` | 32 / 42 / -0.3 | 章节首页（如"血液检测"） |
| `headline` | 24 / 32 / -0.2 | 卡片主标题、对话标题 |
| `title` | 18 / 26 / 0 | 列表项主行 |
| `body` | 15 / 24 / 0 | 正文，所有日记 / 备注 |
| `body-sm` | 13 / 20 / 0 | 次要描述 |
| `label` | 12 / 16 / +0.6 | tag、caption、表单 label — **必须 tracking +0.6** |
| `mono` | 14 / 20 / 0 | 数值、剂量、HRT 第 N 天 |

### CJK 排版细节
- 中日文段落首行 **不缩进**（杂志风，区别于报纸）
- 中文标点用全角，但在标题中可选半角避免大空白
- 行高对中文按 1.7 起步（v1 没规定，导致中文段落显得局促）

## 5. Layering 哲学

**不靠投影、不靠边框、不靠渐变，只靠"纸的厚度"。**

```
[bg #F4F1EA  ← 月白底，全屏铺]
  └─ surfaceContainerHigh #EAE6DD  ← 杂志的章节间隔块，宽窄不一
       └─ surfaceContainerLowest #FBF8F2  ← 一张稍微抬起的"内文页"
              └─ ink + accent  ← 内容
```

关键视觉语法：

1. **以"段落"为视觉单位，不是"卡片"。** v1 是卡片堆砌；v2 是版面排印。一个 surfaceContainerLowest 里可以放 1 个标题 + 3 段文本 + 1 张图，**不再切成 3 张小卡**。
2. **块面边界靠色阶差 4-6 个明度单位**（如 #F4F1EA → #FBF8F2 是 6 步）。这刚好是人眼"察觉但不刺眼"的阈值。
3. **绝不投影**（继承 DESIGN.md v1 唯一保留的原则）。
4. **AppBar 区域不用 BackdropFilter blur**（v1 在 13 处文件里手写 blur 12px，是工程债也是审美错位）。改用 `surfaceContainerHigh` 实底 + 滚动时下方加 1px `outline @ 30%` 细线（且仅这一处允许实线）。
5. **照片不切圆角 < 4px**。杂志里照片要么直角、要么圆 2px 暗示纸感，**绝不 16px 软糖角**。

## 6. 形状与节奏

### 圆角策略
- 卡片 / surfaceContainer：**4px**（v1 是 16-24px）
- 输入框：**2px**
- 按钮：**6px**（不再用 stadium full 圆角，那是 Web2.0 + Material 的痕迹）
- 头像 / 避免硬边的小图标：圆形（保留）
- 照片：**0px** 或 **2px**

### 间距网格

底层 4px 基线，但 v2 故意**跳过 12 和 20**，强制设计师在大尺度上做选择：

| Token | Value | 场景 |
|---|---|---|
| `xs` | 4 | 行内 icon ↔ 文本 |
| `sm` | 8 | 紧密元素之间 |
| `md` | 16 | 段落内、卡片 padding |
| `lg` | 32 | 段落之间、章节起始 |
| `xl` | 64 | 章节之间、Today 顶部留白 |
| `xxl` | 128 | 屏幕首屏到第一个内容块（Onboarding 用） |

**规则：相邻两个间距必须跨级**（不允许 16 紧挨 16），强制留白节奏。

### 不对称美学

杂志栅格永远不对称。明确允许这些位置偏移：

- **顶部 hero 标题左对齐到 24px，但右侧给 96px+ 留白**（不像 v1 居中或两端对齐）
- **章节标题左侧加 4px 黛蓝竖线，长度只覆盖标题首行**（不是分隔符，是"段落标记"）
- **一屏只有一个视觉重心**，重心位置允许偏左 30% 或偏右 30%，但绝不居中
- **列表项左侧留 32px 永久留白**用于偶尔出现的章节序号（如"01 / 用药"）

## 7. 文案调性（Tone of Voice）

**v1 调性**：温柔守护者，第二人称鼓励，emoji 装饰（💊、✅、🌸）
**v2 调性**：克制、第三人称视角、像编辑写图说，**完全去 emoji**

| 场景 | v1 当前 ARB | v2 编辑级东亚 |
|---|---|---|
| 服药 CTA | `takeDose: "服药"` / `reminderNotifBody: "{drugName} {dosage}{unit} — 该吃药啦 💊"` | `takeDose: "记一次"` / `reminderNotifBody: "{drugName}　{dosage}{unit}　到时间了。"`（全角空格 + 中文句号收尾，无 emoji） |
| 时间线空态 | `noTimelineEvents: "暂无时间线事件"` | `noTimelineEvents: "本期空白。\n你今天还没有记录。"`（双行，第一行像杂志 stand-first，第二行才解释） |
| 删除确认 | `confirmDeleteDrug: "确定删除这种药物吗？此操作不可撤销。"` | `confirmDeleteDrug: "删除后不可恢复。\n仍要继续？"`（先说后果，再问意愿；句末不用问号叹号，问号即可） |
| 庆祝（替代花瓣） | （动画式） | 文案出现一行：`recordFooter v2: "今日已记。"`（四个字，黑色宋体，1.2 秒淡入淡出，无粒子） |
| 每日 quote | `dailyQuote0: "你的每一次坚持，都会在未来悄悄开花。"`（甜） | `dailyQuote0 v2: "今日宜，按时。"`（黄历体，三五字，留下解读空间） |

**情感峰值不靠装饰，靠克制中的转折**。当 v1 用花瓣粒子说"你做到了！"，v2 让屏幕静下来一秒，用一行黑底白字宋体说"今日已记。"——后者更有重量感，且更适合一个"陪你十年的健康记录"。

## 8. 参考产品（5 个，给具体来源）

1. **MUJI Notebook（无印良品手帐 app）** — 启发于"内容即装饰，不另加装饰"的留白哲学；首屏只有日期 + 一个写入入口。来源：MUJI 官网 / App Store JP 区"MUJI passport"姊妹应用截图。

2. **Kinfolk 杂志数字版（kinfolk.com）** — 启发于"一段一图一空"的栅格节奏，特别是它的章节首页：超大标题左下，正文从屏幕中部才开始。来源：https://www.kinfolk.com/journal/

3. **Apple Watch 照片表盘（watchOS）** — 启发于深色模式下的"无装饰背景 + 一抹白字时间"的克制美学。HanaNote 的 Today Tab 顶部就是这种感觉。来源：Apple Watch 表盘库 - "Photos"。

4. **新潮文库 装幀（新潮社文庫）** — 启发于纸本小说封面：标题宋体竖排或左对齐，副标题极小，一抹色块（多为深蓝或朱砂）。来源：日本亚马逊"新潮文庫"分类页缩略图阵列；典型示例：村上春树《ノルウェイの森》文庫版。

5. **山本耀司 Y-3 / Yohji Yamamoto 官网** — 启发于"全屏黑/全屏米色 + 一行小字"的奢侈感叙事，证明"少"可以等于"贵"。来源：https://www.yohjiyamamoto.co.jp/

> 反例（明确不参考）：TRACE（紫粉温柔）/ Clue（圆环 + Gen-Z 渐变）/ 任何 Material You 动态色

## 9. 应用到 Today 屏幕

**v2 的 Today 像一份每日报纸的头版**：

- **顶部 hero（占 25% 屏高）**：左对齐 32px。第一行是 `display-xl` 宋体的「早安」（从 ARB `goodMorning` 读，但去掉所有 emoji）。第二行是 `body-sm` 淡墨 `#5E5A52` 的当天日期 + HRT 第 N 天，等宽数字 mono。**右上角无任何按钮**（设置入口下沉到 Profile Tab）。

- **今日清单（占 50% 屏高）**：不是卡片列表，是杂志的"今日要目"。每条用药一行：左侧 4px 黛蓝竖线（已服药变成淡墨），中间药名（`title` 宋体）+ 剂量（`mono`），右侧时间（淡墨 `body-sm`）。**条目之间只有 24px 间距，无分隔线**。

- **服药状态表达**：未服 = 黛蓝竖线 + 黑墨标题；已服 = 淡墨竖线 + 淡墨标题（整体褪色，但保留可读性 AA）。**绝不打勾、不变绿**——勾是工具语言，褪色才是日记语言。

- **庆祝反馈**：点击"记一次"后，屏幕静默 0.4 秒（不立即 dismiss），淡入一行黑墨宋体 `今日已记。` 居中，1.2 秒淡出，无声、无粒子、无震动（震动可作为可选项）。仪式感来自"等待"而不是"爆发"。

- **底部**：底栏 5 个 Tab，仅文字标签，无图标。Active Tab 用 `accent` 黛蓝下划线 2px。这是杂志"目录页"的语法。

## 10. 应用到 Onboarding 屏幕

**v2 的 Onboarding 是一本书的扉页**，不是欢迎弹窗。

- **第 1 屏（欢迎）**：全屏月白底，屏幕中央偏上 30% 处一行 `display` 宋体「HanaNote」（黑墨），下方一行 `body` 淡墨「私人健康内刊。仅你可见。」（不用 v1 的`onboardingWelcomeSub: "你的私密 HRT 健康记录空间"`，那句太长太软）。底部一个 6px 圆角 `accent` 黛蓝按钮「翻开」（替代 `onboardingNext: "下一步"`），按钮宽度只占 50% 屏宽，居中。

- **第 2 屏（取名）**：左对齐 32px，宋体大标题「你想被怎么称呼。」（无问号 — 这是杂志 stand-first 风格）。下方 96px 留白后才是输入框。输入框无外框，只在底部一条 1px `outline` 烟灰线，focus 时变 `accent` 黛蓝 2px。占位符 `body` 淡墨「随便写。」。

- **第 3 屏（HRT 起始日）**：标题「从哪一天起，你不再是从前的自己。」（这句是 v2 的灵魂——v1 是 `onboardingSetHrtDate: "你的 HRT 起始日是？"`，太工具）。下方一个简洁日期块：超大 `display-xl` 宋体显示选中日期，点击进入日历。

- **第 4 屏（添加药物）**：直接列出 22 种 HRT 模板，每个一行（不是图标网格）。这是杂志的"目录页"。

- **第 5 屏（完成）**：单行宋体「准备好了。」+ 「开始」按钮。**没有 confetti，没有花瓣**。

**核心转变**：v1 的 Onboarding 是"教程 + 鼓励"，v2 是"一本书的目录与扉页"。用户读完后会觉得"这个 app 把我当成人在对话"。

## 12. 风险与权衡

### 风险 1：原 v1 用户觉得"冷漠"，失去温度感
- **缓解 1**：温度藏在文案里。每日 quote 改成黄历体短句（"今日宜，按时。"），比甜文案更有"被读懂"的颗粒感。
- **缓解 2**：庆祝反馈虽不爆发，但保留"今日已记。"的仪式静默，配合可选触觉反馈，比花瓣更有"重量"。
- **缓解 3**：在 Profile Tab 保留一处暖色——头像默认色用 `accentMuted` 远黛 + 用户首字母宋体，这是一个"有人陪你"的小锚点。

### 风险 2：需要更高质量的字体授权
- 思源宋体 / 思源黑体 / Noto Serif JP / Noto Sans JP / Inter / Spectral / JetBrains Mono **全部 SIL OFL / Apache 2.0 开源免费**，可商用。**0 授权成本**。
- **缓解**：APK 体积会增加约 5-8MB（CJK 字体子集化后）。Web 端用 Google Fonts CDN 异步加载。这是可接受的代价。

### 风险 3：CJK 排版高度依赖字体细节，工程成本增加
- **缓解 1**：建立 `HanaTypography` 中央 token（v1 缺这一层），所有字号 / 行高 / tracking 一处定义，21 屏统一受益。这本身就在 audit 的 P1 ranking 里。
- **缓解 2**：思源宋体的中日文字形完全统一（Pan-CJK 设计），中日双语切换不用换字体堆栈。
- **缓解 3**：先实现 Phase 2.1 的 8 个 v1 共享组件（HanaCard / HanaEmptyState 等），新设计语言降级渗透，不一次性重写 21 屏。

### 风险 4：Web 端 CJK 字体首屏加载延迟
- **缓解**：用 `font-display: optional` + 系统宋体 fallback（Mac: STSong, Win: 宋体, iOS: 苹方-繁宋），首屏 0 闪烁，二屏再切换思源宋体。

## 13. 适合 / 不适合

### ✅ 适合
- **22-35 岁的成熟用户**，已过青春期审美 turbulence，开始欣赏「克制」
- **东京 / 京都 / 上海 / 台北 城市文化敏感用户**，能识别杂志、独立书店、选物店的视觉语言
- **想要"健康记录但不想感觉自己是个病人"** 的用户——v2 把医疗感彻底洗掉，留下"日记感"
- **重度记录型用户**——杂志版面比卡片更适合长文日记 + 多张照片混排
- 长期持有 5+ 年的"陪伴型 app"定位（这个调性不会过时）

### ❌ 不适合
- **16-20 岁早期 HRT 用户**，刚出柜需要的是肯定 + 温暖 + 情绪共鸣，v2 的克制可能被读成"冷淡"或"不在乎我"
- **Gen-Z 重度二次元审美用户**，期待粉色 / 渐变 / 表情包式装饰
- **想要被"哄"的用户**——v2 不会哄，它会"敬"
- **东南亚部分子市场**，当地审美主流偏暖色高饱和，v2 单色 + 黛蓝可能显得"性冷淡"或"不像 app"
- 喜欢 TRACE/Hormone Helper 紫粉系的用户群体

## 14. 决策推荐力度

**8.5 / 10**

理由：v2 是 HanaNote 在"中日东亚 16-35 MTF HRT 整合工具"赛道上**最难被复制、最贴近"成人陪伴十年"愿景**的方向。它放弃了 16-20 岁子市场，但拿到的是一个**没有任何竞品占领过的位置**——竞品分析里 TRACE 紫粉、Clue 圆环、Trans Memo 日系极简都在做减法工具，**没有人做过"杂志感的健康记录"**。

扣 1.5 分的原因：
- 风险 1（冷漠感）真实存在，需要文案 / 触觉 / 一抹强色三处持续校准；
- 它需要工程团队接受"重写而不是渐进"——v1 的 13 处 BackdropFilter / 圆角 16 / 渐变 token 都要拆，短期 LOC 删除会比新增多。

但作为方向 A 的提案者，我认为这是**最有作品级野心的一条路**。如果用户选 B（科技隐私）会得到一个酷但冷的产品，选 C（治愈手账）会得到一个甜但同质化的产品，选 A 会得到**一份十年后翻出来仍不过时的私人内刊**。
