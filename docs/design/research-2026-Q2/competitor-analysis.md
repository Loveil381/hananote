## 1. 竞品逐个分析

### 竞品 A：TRACE
- **目标用户**：跨性别社区（含 MTF/FTM/NB），偏北美/欧洲英语圈青少年与年轻成人。`[领域知识]`
- **价值主张**：HRT 旅程的"成长记录册"——拍照对比 + 里程碑庆祝 + 小型同伴社区。`[已记录]`
- **设计语言**：暖紫/粉紫主导的"温柔友好"调性，圆角卡片，插画化空间状态，整体偏 Instagram-lite。`[领域知识]`
- **关键屏幕（描述）**：
  - 登录：第三方账号 + 邮箱注册，社区导向。`[领域知识]`
  - 首页：Timeline / 里程碑摘要卡片。`[已记录]`
  - 数据录入：拍照为主，剂量/心情为辅，提醒以日历卡形式呈现。`[已记录]`
  - 数据查看：照片对比滑块 + 月份回顾。`[领域知识]`
- **文案调性**：第二人称、鼓励式（"You did it!" / "1 month on E"），Affirming 词汇，去医疗化。`[领域知识]`
- **盈利模式 + 隐私态度**：免费 + 可能内购，云同步意味数据上传第三方服务器；隐私表述未必有端到端加密承诺。`[空白：实际订阅价格、数据政策、是否 E2EE]`
- **HanaNote 可借鉴 / 警惕**：
  - 借鉴：里程碑庆祝的情感峰值设计（HanaNote 已有花瓣粒子，方向一致）。`[已记录]`
  - 警惕：社区功能引入身份暴露风险，与 HanaNote"私密空间，非社交平台"的愿景冲突。

### 竞品 B：Hormone Helper
- **目标用户**：北美英语圈 MTF，偏向有医生跟进的"医患沟通"型用户。`[领域知识]`
- **价值主张**：HRT 追踪 + 医生沟通模板 + 拍照 + 心情追踪。`[已记录]`
- **设计语言**：清爽白底 + 蓝/紫强调色，偏"轻医疗"风格，比 TRACE 更工具感、更少社交化。`[领域知识]`
- **关键屏幕（描述）**：
  - 登录：本地账号或 iCloud 同步。`[领域知识]`
  - 首页：今日用药 + 最近心情/照片。`[领域知识]`
  - 数据录入：表单驱动，分项明确（剂量/方式/部位/心情）。`[领域知识]`
  - 数据查看：可导出给医生的报告页面。`[已记录]`
- **文案调性**：中性、信息密度较高，偏临床描述。`[领域知识]`
- **盈利模式 + 隐私态度**：iOS Only，订阅或一次性买断；iOS 沙盒原生隐私，但跨设备同步走 iCloud。`[空白：定价、是否使用第三方分析 SDK]`
- **HanaNote 可借鉴 / 警惕**：
  - 借鉴：医生沟通模板/可分享报告的概念（HanaNote PDF 报告 v1.1.0 缺口正好对应）。`[已记录]`
  - 警惕：iOS-only + 英语单语，无法服务东亚用户，HanaNote 的多语言+Android+Web 是结构性优势。

### 竞品 C：Trans Memo
- **目标用户**：日本/东亚 Android 用户，偏向"我只想记得吃药"的极简主义者。`[领域知识]`
- **价值主张**：用药 + 库存提醒，单一专注。`[已记录]`
- **设计语言**：日系简约 Material，留白多，色彩克制（推测为浅色系单色调）。`[领域知识]`
- **关键屏幕（描述）**：
  - 登录：可能无登录，纯本地。`[领域知识]`
  - 首页：今日服药清单 + 库存预警。`[领域知识]`
  - 数据录入：极简表单，少字段。`[领域知识]`
  - 数据查看：用药历史列表。`[已记录]`
- **文案调性**：日语为主，简洁、礼貌、去情感化。`[领域知识]`
- **盈利模式 + 隐私态度**：免费 + 可能广告；纯本地数据隐私性较好，但加密程度未知。`[空白：是否有云同步、广告 SDK 情况]`
- **HanaNote 可借鉴 / 警惕**：
  - 借鉴：库存追踪是 HanaNote 当前明确缺口（PRODUCT-VISION 未列）。`[已记录]`
  - 警惕：单一功能在用户增长曲线后段会被整合型产品替代——这正是 HanaNote 的差异化方向。

### 竞品 D：estrannaise.js
- **目标用户**：技术敏感的 HRT 极客（reddit r/TransDIY 等社区），关心 PK 模型、酯型差异、剂量优化。`[领域知识]`
- **价值主张**：开源、科学严谨的 3 室 PK 模拟器 + MCMC 拟合 + 多酯型支持。`[已记录]`
- **设计语言**：纯 Web、深色 + 高对比度、图表为绝对主角，工程师审美——更接近"可视化论文"而非消费产品。`[领域知识]`
- **关键屏幕（描述）**：
  - 登录：无（纯静态页面）。`[领域知识]`
  - 首页：曲线 + 参数控制面板二分屏。`[领域知识]`
  - 数据录入：剂量/酯型/时间点输入框。`[领域知识]`
  - 数据查看：交互式曲线 + 不确定性带（MCMC 后验）。`[已记录]`
- **文案调性**：英文，技术性强，假设读者懂药代动力学术语。`[领域知识]`
- **盈利模式 + 隐私态度**：开源免费，纯客户端运算，隐私结构最优（数据不离开浏览器）。`[已记录]`
- **HanaNote 可借鉴 / 警惕**：
  - 借鉴：模型严谨度——HanaNote V2 + Hana-PK 双引擎已对标。`[已记录]`
  - 警惕：工具感 UI 对 16-25 岁初级用户过于劝退，HanaNote 需做"科学引擎包温柔皮肤"的翻译层。

### 竞品 E：TFS Simulator (Transfeminine Science)
- **目标用户**：注射 E2 用户中关心血药浓度的子集。`[领域知识]`
- **价值主张**：单一注射酯型的 PK 偏移叠加模拟。`[已记录]`
- **设计语言**：学术站点风格，浅色背景 + 单一交互曲线，无品牌视觉。`[领域知识]`
- **关键屏幕（描述）**：单页应用，输入 + 曲线一屏完成。`[领域知识]`
- **文案调性**：科普式英文，比 estrannaise 更面向大众但仍学术。`[领域知识]`
- **盈利模式 + 隐私态度**：非营利学术资源，无盈利动机；纯客户端。`[领域知识]`
- **HanaNote 可借鉴 / 警惕**：
  - 借鉴：作为"权威背书"参照（HanaNote 已超越其简单偏移模型）。`[已记录]`
  - 警惕：仅注射、仅一个酯型——HanaNote 不应回退到这一窄范围。

### 竞品 F：ShotTraX
- **目标用户**：iOS 注射 HRT 用户（含 MTF 与 TRT），偏注重"打针仪式感"的人群。`[领域知识]`
- **价值主张**：注射点位轮换追踪 + 简易社交分享。`[已记录]`
- **设计语言**：医疗工具风（人体图示、解剖示意），蓝色冷调为主。`[领域知识]`
- **关键屏幕（描述）**：
  - 首页：上次注射点 + 下次建议点位。`[领域知识]`
  - 数据录入：人体图上点选注射部位。`[领域知识]`
- **文案调性**：技术性、男性化倾向（受 TRT 用户基础影响）。`[领域知识]`
- **盈利模式 + 隐私态度**：iOS App Store 标准模式。`[空白：定价、数据政策]`
- **HanaNote 可借鉴 / 警惕**：
  - 借鉴：注射点位轮换的视觉化 UI 模式（HanaNote 当前可能未充分可视化）。`[空白：HanaNote 是否已有点位轮换 UI]`
  - 警惕：冷调 + 男性化倾向不适合 HanaNote 二次元温暖审美。

### 竞品 G（外延）：Clue（月经周期追踪）
- **目标用户**：18-35 岁女性，全球范围，强 Gen-Z 设计偏好。`[领域知识]`
- **价值主张**：科学严谨的周期追踪 + 数据可视化 + 多元包容（覆盖 trans/non-binary）。`[领域知识]`
- **设计语言**：高识别度的红/粉/紫渐变 + 大圆环主图 + 极简白底，被广泛视为"健康记录类设计语言"的事实标准之一。`[领域知识]`
- **关键屏幕（描述）**：首页 = 大圆环显示当前周期阶段，下方为分类记录卡。`[领域知识]`
- **文案调性**：包容、科学、第二人称、不卖焦虑。`[领域知识]`
- **盈利模式 + 隐私态度**：免费 + Clue Plus 订阅；隐私政策 GDPR 友好但仍上传服务器。`[领域知识]`
- **HanaNote 可借鉴 / 警惕**：
  - 借鉴：圆环主图 + 大留白 + 包容性文案是品类标杆。
  - 警惕：Clue 的设计语言已被广泛模仿，HanaNote 若简单复刻会陷入"又一个健康追踪 App"的同质化。

### 竞品 H（外延）：Apple Health / MyChart
- **目标用户**：全龄段，偏临床/医疗机构对接。`[领域知识]`
- **价值主张**：跨设备健康数据聚合（Apple）/ 医院 EHR 患者端（MyChart）。`[领域知识]`
- **设计语言**：Apple Health 偏白底 + 类别色块 + 大数据卡；MyChart 老派蓝色医院风。`[领域知识]`
- **HanaNote 可借鉴 / 警惕**：作为"医疗严肃端"参考，HanaNote 应明确**不**走这条路——目标用户对医院系统普遍有距离感。

## 2. 横向模式分析

### 主流设计语言（按维度归纳）
- **色调**：HRT 直接竞品两极分化——温暖向（TRACE 紫粉、Hormone Helper 蓝紫）vs 工具向（estrannaise 深色、ShotTraX 医蓝）。**暖色 + 紫粉色系**是面向情感连接的事实主流。
- **医疗 vs 生活**：除 estrannaise/TFS/ShotTraX 外，主流均向"生活感"倾斜——拒绝医院蓝白、拒绝处方单式表单。
- **图形 vs 数据**：消费向产品（TRACE/Hormone Helper）以"卡片 + 摘要数字"为主，数据向产品（estrannaise/TFS）以"曲线 + 参数面板"为主。**HanaNote 同时背负两种诉求**，这是结构性张力。
- **首页结构**：今日聚合（用药清单 + 心情/照片快记）几乎是品类公约数。
- **数据录入**：分类 Tab + 表单字段，少有真正创新；快速记录（FAB / 长按 / Quick Action）尚未成为主流。

### 主流文案调性
- 第二人称、肯定式（"You're doing great"）、去医疗术语化是英文圈共识。
- 日语圈（Trans Memo）偏礼貌克制、信息密度高、情感留白。
- 中文圈在此品类**几乎是空白**——HanaNote 有定义中文 HRT 工具语调的先发位。

### 主流交互模式
- 拍照对比是 HRT 品类独有"杀手级"交互（TRACE/Hormone Helper），HanaNote 已具备但 UI 抛光程度未知。
- 提醒系统普遍依赖系统通知，少见应用内"温柔催办"创新。
- PK 模拟器是 HanaNote 在消费向产品中**罕见**的差异化武器——TRACE/Hormone Helper 都没有。

## 3. 机会缺口

> 候选差异化维度，每条均有"空间无人占领"或"现有方案薄弱"的依据。

1. **中日东亚审美的 HRT 工具**：直接竞品全部为英语圈或日语单语简约工具，**面向东亚 16-25 岁的二次元温暖审美 HRT 整合工具是空白市场**。这与 HanaNote 已有的樱花花瓣等元素一致。
2. **科学引擎 + 温柔皮肤的翻译层**：估算工具（estrannaise/TFS）严谨但劝退、消费工具（TRACE/Hormone Helper）温柔但模型粗糙，**两者交集是空地**。HanaNote 双引擎已具备技术底座，待设计语言把"PK 曲线"翻译成普通用户能看懂的视觉表达。
3. **隐私即设计语言（Privacy-as-Aesthetic）**：现有竞品要么云同步默认（TRACE/Hormone Helper），要么纯静态无功能（estrannaise）。**让"加密、伪装、紧急清除"成为视觉与文案上可感知的安全感（而非埋在设置深处的开关）**是空白。
4. **跨身份阶段叙事**：现有产品以"今日"为主，**长时间维度的"成长册"叙事**（半年/一年/全程对比、节气式里程碑、可导出给未来自己的时间胶囊）尚未被认真做过。
5. **Web + Android 双形态而非 iOS 优先**：直接竞品有 4/6 是 iOS 或 iOS-first。HanaNote 的 Android + PWA 主战场天然贴近东亚 + 东南亚低预算用户，设计语言可面向**非 Apple 美学体系**做差异化。

## 4. 信息空白

建议团队补充以下信息以提升后续阶段决策质量：

1. **TRACE / Hormone Helper / Trans Memo 的实际 UI 截图**——COMPETITORS.md 仅有功能列表，无视觉证据。建议截图归档至 `docs/design/research-2026-Q2/competitor-screenshots/`。

3. **隐私政策细节**：TRACE 是否使用 Firebase/Mixpanel、Hormone Helper iCloud 同步是否端到端、Trans Memo 是否上传任何数据。HanaNote 的"极致隐私"主张需要这些参照系来量化优势。
4. **用户量与活跃度信号**：App Store/Play Store 评分数、reddit r/asktransgender 提及频率。判断哪些是真实主流、哪些是营销噪音。
5. **东亚直接竞品补缺**：中文/简中圈（如小红书或微信生态内的轻量 HRT 工具）、韩国市场。COMPETITORS.md 完全未覆盖中文圈，可能存在未识别的本土对手。
6. **HRT 之外的相邻空间**：Flo（月经追踪头部）、Ovia、Maven Clinic 的设计语言是否值得列为参考。当前外延仅举 Clue/Apple Health。
7. **HanaNote 自家点位轮换 UI 现状**：相对 ShotTraX 的对比基准在团队内部是否清晰。

## 5. 给阶段 0 的输入清单

按优先级推荐 Phase 0 重点参考的 5 个产品：

1. **TRACE** —— 直接同品类、温暖审美、里程碑设计的最近邻参照，建议作为"调性北极星"采样对象。
2. **Clue** —— 健康追踪品类的设计语言事实标杆，圆环主图 + 包容文案是必读教材；用作"避免同质化"的对照基准。
3. **estrannaise.js** —— 科学严谨度的对照系，提醒 HanaNote 不丢失数据深度；Phase 0 需思考"如何把它的曲线翻译为温柔语言"。
4. **Hormone Helper** —— 医生沟通模板/可分享报告的最佳现成实现，对应 HanaNote v1.1.0 PDF 报告缺口的设计输入。
5. **Trans Memo** —— 唯一的日语圈直接竞品，作为日语市场设计语言的语感锚点。

> 次优先级：ShotTraX（仅在做注射点位 UI 时参考）、Apple Health/MyChart（仅作为"我们不要变成什么"的反面教材）。
